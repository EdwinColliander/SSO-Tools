SAML Inspector

A browser-based toolkit for troubleshooting SAML 2.0 federations. No data leaves your machine.

Metadata →

Fetch or parse SP/IdP metadata XML. Useful for verifying that endpoints, bindings, NameID formats and certificates are configured correctly on both sides of a federation. Certificate details (CN, SAN, expiry) are inspected automatically.

Endpoint mismatch Certificate expiry Missing SLO NameID format mismatch
SAML Request →

Parse an AuthnRequest — either raw XML or Base64-encoded. Shows all relevant fields: ID, issuer, destination, ACS URL, NameID policy and binding. Useful for verifying that the SP is sending the correct request to the IdP.

Wrong destination URL Wrong ACS URL NameID format mismatch Binding mismatch
SAML Response →

Parse a SAML Response and its embedded Assertion. Extracts status, subject, conditions, attributes and verifies the XML signature using the certificate embedded in the response. Useful for diagnosing failed logins at the SP side.

Signature invalid Wrong audience Missing attributes Expired assertion Wrong NameID
Decode / Encode →

Decodes or encodes the SAMLRequest/SAMLResponse transport layer. Redirect Binding compresses the XML with raw DEFLATE before Base64-encoding it — paste the query parameter value directly and decode to inspect the underlying XML. POST Binding uses Base64 only.

Inspect requests from browser Read URL-encoded SAMLRequest Craft a manual test request
Certificate →

Inspect an X.509 certificate in detail. Paste raw Base64 or a full PEM block. Shows subject, issuer, SAN, validity period, key type and SHA-1/SHA-256 fingerprints. Useful for comparing the certificate in metadata against the one actually used for signing.

Fingerprint comparison Expiry check SAN/CN verification Key size audit
